ROS初步-功能包package编译的两种方式

发表于 2019-06-01 | 分类于 ros | | 阅读次数:

ROS的包编译有两种方法。一种是用rosmake，这种方法简单；另一种是用catkin_make，这种方法更方便包的管理和开发。这两种方法都是先建立工作空间workspace（类似于vs下的解决方案，用来管理很多的项目），然后建立包package（类似于vs下的项目），最后利用rosmake或者catkin_make进行编译和运行。学会第一种方式，再去学习第二种就很简单了。

可以看到，节点是可执行程序，要生成可执行程序自然需要 add_executable 这是cmake的东西。同时就像编写普通的c/c++程序一样，当程序要用调用某个库，自然需要在链接的时候提供链接库，cmake中用target_link_libraries来指定要生成某个可执行文件所需要用到的链接，所以我们这里有两个可执行文件，他们都有 #include <ros/ros.h> ，所以需要target_link_libraries指明链接哪个库。

可是提供了只这个库的名称，cmake并不能顺利编译，这个链接库的路径要让cmake找到，还需要include_directories,cmake编译完链接时在gcc库找不到会去找include_directories提供的链接库。
